Apacheの設定ファイル(http.conf)のそこんところ

Apache

WebサーバーのApacheの設定ファイル、「httpd.conf」を見ていきます。

そこんところブログらしく、あっちこっちも見て行きません。とりあえず気になったところを

ServerRoot

Apacheをインストールした場所のパス。以降のディレクトリ指定で相対パスを利用する場合等に利用

以降の出てくるファイルのパス表記では、このディレクトリを起点とする相対パス表記されているものがチラホラ。基本デフォルトのまんまかな？

Listen

特定のIPアドレスやポート番号だけを受け付ける指定。VirtualHostではここの設定のポート範囲内のみ使用できる

LoadModule

ApacheにはWebサーバーとしてのコア機能以外にも、認証や圧縮転送など様々な付加機能がそれら付加機能をApacheはモジュールとして提供されており、選択可能なモジュール構造で様々なクライアントからのアクセスでも適材適所な機能を提供することが出来ます。

現在使用されているモジュールを確認するには、

各モジュール名の後ろには(static)または(shared)という記載があります。
Apacheのモジュールはstatic(静的)なものと、shared(動的？)なものに分けられます。Apacheのインストール時から組み込まれているstaticモジュールはモジュールの使用の有無にかかわらずApacheに組み込まれ続けるもの、一方sharedモジュールは設定ファイルで有効無効を切り替えられるます。

DocumentRoot

ウェブサーバー(これ、Apache)がクライアントからのリクエストに対して応答するコンテンツを格納する場所を記述します。

ルートディレクトリを指定しているので、例えばWebサーバーを持っている端末から「http://localhost/」でアクセスした結果は、このディレクトリ内のindex.htmlが表示されます。

DirectoryIndex

上記の通り、「http://localhost/」でアクセスした場合はDocumentRoot内のindex.htmlをレスポンスとして返します。

ファイル名を指定しない(ディレクトリ名のみの)場合に返すファイルの名前をDirectoryIndexで指定します。index.html以外が良い場合はここを変更する。

IfModule

タグ内に書かれたモジュールが読み込まれた時にタグで挟まれた設定、処理を行われます。

Directory

タグ内で書かれたディレクトリの範囲のみに設定を反映させることが出来ます。

AllowOverride

上位のディレクトリで設定したものを下位ディレクトリで変更することを許可するかを記述します。AllowOverride None の場合は、下位ディレクトリの設定で変更することはできません。一方、AllowOverride All の場合は変更することができます。

パスを通す？環境変数？....bashファイル？

Unix/Linux

初めてのMacを手にしたのは、初めて配属先に来たとき。
そして初めて開発機としての環境構築なるものを行ったわけですが、その時先輩がやたら言っていたのが、「パスを通す」、「バッシュファイル」。

バスケシューズ履いてパスするのかよ。ファイルの名前ってことは分かるけれど・・・・・そんな思い出を思い出したので改めて「MacのPATHについて」整理します。

環境変数

システムの実行ファイルが置いてあるディレクトリの位置を設定し、動作するアプリケーションソフトが参照するためのものが記述されている。

PATHを通す

これは実行ファイルがある場所をPCに教えて上げること。ディレクトリの奥底にある実行ファイルを使うためにいちいちそこまでのパスを書くのは・・・・ありえない。簡単にコマンドを実行するために覚えてもらう（PATHを通す）わけですね。

設定ファイルの場所

  • 全ユーザー対象のPATH

    「/etc/paths」に記載されています。確認してみると・・・

    「/usr/local/bin」「/usr/bin」「/bin」「/usr/sbin」「/sbin」「/opt/X11/bin」の５箇所に実行ファイルが置かれているようです。

  • ユーザー単位のPATH

    ホームディレクトリ内の「.bash_profile」に書き込みます。

    存在しない場合があるので、ない場合は作っちゃいます。

基本的にはこの「~/.bash_profile」に(ユーザー単位に)環境変数を設定した方がいいらしい。

SQLのデータ型

データベースを実際にいじる前にさらっておこう。

  1. 数値型
    • int
      • 他に「tinynt」「smallint」「mediumint」
    • float
    • double

  2. 文字列型

    (M)は最大文字数を入力する。

    • CHAR(M)
      • M個文字を格納出来る固定長の文字列。
      • 指定文字数に対して余った分にはスペースが入る。
      • Mを記入しない場合は1文字の文字列。
    • VARCHAR(M)
       M個文字を格納出来る*可変長の文字列。
      • Mは0～255の範囲で指定
      • 格納する文字列の最後にスペースが入っていても削除されて格納される。
    • TEXT
      • 最大長が 65535バイトの可変長文字列。
      • テキストデータを格納する。
    • BLOB
      • バイナリデータを格納する。
